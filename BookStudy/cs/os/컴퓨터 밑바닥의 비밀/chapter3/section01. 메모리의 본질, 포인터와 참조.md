# 저수준 계층? 메모리라는 사물함에서부터 시작해 보자
- CPU는 메모리가 있어야 동작 할 수 있다.

## 메모리의 본질, 포인터와 참조.md
- 메모리 파생 개념에 대해 알아보자.
  1. 바이트(byte)
  2. 구조체(structure)
  3. 객체(object)
  4. 변수(variable)
  5. 포인터(pointer)

### 3.1.1 메모리의 본질은 무엇일까? 사물함, 비트, 바이트, 객체
- 메모리는 사물함으로 비유할 수 있고 원리가 본질적으로 다르지 않다.
  - 각각 고유 번호를 갖는 여러 개의 독립된 공간
  - 사물함 == 메모리 셸(memory cell)
  - 메모리 셸(1bit)
    - 0, 1만 보관 가능 -> 1bit, 즉 1bit은 0 또는 1
    - 1bit -> yes or no, true or false -> 대비되는 두 가지 정보만 표현 가능
    - 1bit 8개 -> 1byte (8개의 사물함)
  - 바이트
    - 각 byte마다 번호를 붙인다. -> 메모리 주소
    - 메모리 주소는 특정 메모리 셸(사물함) 여러 개를 찾을 수 있다.
    - 1byte -> 정보 표현 능력에 한계
    - 일반적으로 4byte를 묶어 하나의 정수 표현 단위로 사용한다.
  - 구조체 & 객체
    - 12byte를 사용해 정보 조합
- 정리
  - 1bit -> 0과 1일 뿐
  - 1byte -> 8bit 묶어 사용
  - 구조체 또는 객체 -> 12byte 묶어 사용

### 3.1.2 메모리에서 변수로: 변수의 의미
- 8byte 메모리 주어졌다고 가정, 어떻게 읽고 쓸까?
  - 메모리는 사물함이라는 사실을 인지해야 한다.
  - 1byte 사물함 8개가 하나로 묶여있는 상태 (순차)
  - 1 + 2 연산
    - 1과 2를 메모리에 저장 -> CPU는 메모리에서 값을 읽어 레지스터에 저장해야 연산 가능
    - 숫자 -> 1byte -> 각 사물함 사용 -> 2byte 사용 -> 6byte 여분
    - 메모리 주소에 값을 저장하고 읽어야 한다 -> 메모리 주소를 알아야 한다.
    - 메모리 주소와 숫자의 모호함 -> $ 사용하여 값을 표시, 변수를 사용해 메모리 주소 대체
  - 메모리
    - 메모리 주소 -> 변수 대입 -> 다른 변수에 대입 (b = a) -> b 값에는 a의 값이 들어간다 (메모리 주소 x)
      - 그렇다면 객체를 대입한다면? -> 메모리 낭비, 메모리 부족 현상 발생 가능 (8byte 크기 -> 5byte 객체 -> 다른 변수 대입 -> 10byte 필요)

### 3.1.3 변수에서 포인터로: 포인터 이해하기
- 메모리 주소만 알고 있으면 데이터를 찾을 수 있다.
- 메모리 주소에 숫자가 값으로 해석되지 않고 메모리 주소로 해석된다. -> 포인터 탄생
- 포인터는 주소 자체다?
  - 어셈블리어 수준에만 적용되는 얘기
  - 고급 언어에서는 하나의 변수에 불과하다. -> 저장 값이 메모리 주소일뿐
  - 포인터는 메모리 주소를 더 높은 수준으로 추상화한 것
- 포인터 탄생으로 변수는 값, 메모리 주소 모두를 담을 수 있게 됐다.

### 3.1.4 포인터의 힘과 파괴력: 능력과 책임
- 자바와 파이썬 등 -> 포인터 존재 X 
  - 메모리 주소 직접 노출 X -> 특정 메모리 데이터 직접 조작 불가능
- 직접 메모리 I/O를 하는 것이 모든 상황에 필요하진 않다.
  - 자바와 파이썬 등으로 증명됐다.
    - 포인터를 한 번 더 추상화한 "참조"가 있다.

### 3.1.5 포인터에서 참조로: 메모리 주소 감추기
- 참조란?
  - 별칭 및 별명이랑 비슷하다.
  - 참조 사용시 구체적인 메모리 주소를 얻을 수 없다.
  - 참조와 포인터는 다르다. 다만 참조 사용시 데이터 복제가 필요 없어 동일 효과를 낸다.
- 포인터 -> 메모리 주소 추상화
- 참조 -> 포인터를 한 번 더 추상화
- 메모리 자체 추상화 -> 가상 메모리
- 가상 메모리 지원 시스템
  - 프로세스가 사용하는 메모리 주소 -> 실제 물리 메모리 주소 X
  - 최신 OS -> 기본적으로 모두 가상 메모리 기능 존재

# 49. 타임라인 팬아웃 아키텍처
- 타임라인 구성
  - 쓰기 시 팬아웃 - 게시글 작성시 타임 라인을 미리 구축
    - 팔로우가 많은 경우 쓰기 비용이 많다 -> 쓰기 문제 / 핫키 -> DB 부하 증가시 성능 이슈
  - 읽기 시 팬아웃 - 읽기 시점에 조합
  - 하이브리드 방식
    - 팔로우 수 < 10000 (일반 이용자)
      - 쓰기 시 팬아웃
    - 팔로우 수 >= 10000 (셀럽)
      - 읽기 시 팬아웃

# 50. 타임라인 정렬과 자료구조
- Redis 
  - List
    - LPUSH 앞에 추가 -> O(1)
    - RPUSH 뒤에 추가 -> O(1)
    - LRANGE 범위 조회 -> O(N)
    - LTRIM 범위 유지, 나머지 제거 -> O(N)
  - ZSET (SortedSet)
    - ZADD 점수와 함께 추가 -> O(log N)
    - ZRANGE 순위별 범위 조회 -> O(log(N) + M)
    - ZREMRANGEBYRANK 순위 범위로 제거 -> O(log(N) + M)

# 11. WebFlux의 기본적인 사용법, Flux와 Mono
- Mono
  - 0개부터 1개의 데이터만 방출할 수 있는 객체 -> Optional 등
- Flux
  - 0개 이상의 데이터를 방출할 수 있는 객체 -> List, Stream 등
- Mono를 왜 쓸까?
  - 1개 데이터 가공시 훨씬 가독성 좋고, 유지보수가 수월하다.
- Flux와 Mono 어떻게 사용해야 할까?
  - Flux, Mono는 크게 [데이터 흐름 시작, 데이터 가공, 구독] 흐름
  - Flux는 0개 이상 무한정 데이터 방출 가능
  - Mono는 0개 1개 데이터만 방출 가능

# 16. Scheduler에 대하여
- BoundedElastic
  - 유저가 스레드 요청마다 탄력적으로 생성하여 할당
  - 일정 시간 동안 삭제하지 않고 유지, 재할당
  - 블로킹 작업 처리에 사용
- Parallel
  - 유저가 최초 한 번 호출시 물리 스레드와 같은 양 스레드 생성 후 삭제 안함
  - CPU 작업 병렬 처리시 사용
  - 소수 스레드 운용 -> 블로킹 작업은 피하는게 좋다.

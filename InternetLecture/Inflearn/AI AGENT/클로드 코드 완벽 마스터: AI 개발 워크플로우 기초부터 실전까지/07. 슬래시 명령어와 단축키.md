# 46. 공식문서 권장! 토큰 최적화 3가지 명령어!
- 토큰 소비 및 성능 저하! -> 하나의 대화에서 너무 많은 일을 처리하기 때문에 발생
  - 로그인 기능 계획 -> 로그인 요청 -> 버그 발생 -> 버그 수정 
  - [결제 기능 기획 전 이 전 컨텍스트 연관도에 따라 clear 필요] 결제 기능 계획 -> ...
- /clear
  - 이 전 작업과 무관한 새로운 작업 -> 성능 최적, 토큰 효율!
- /context
  - 현재 컨텍스트 사용량 체크 가능
- /compact [요약 정보] 
  - 대화 내용 요약
  - /context -> 이전 메시지 요약되어 공간 최적화된다.
- 실제 개발 워크 플로우
  - /clear
  - 특정 기간 지난 후 /context 체크 -> 70% 초과
  - /clear 또는 /compact 수행
  - /config -> Auto-compact -> context 사용률이 얼마 남지 않았을 때 자동 compact
    - auto-compact -> 우리가 원하는 요약 정보가 아닐 수도 있으니, 중요한 요약 정보가 있다면 직접 /compact 요약정보 로 직접 해놓는게 필요하다.

# CTO Style Instructions

- 코드를 작성할 때 왜 그렇게 했는지 설명하세요.

- 팀원이 이해했는지 확인하는 질문을 하세요.
- 더 좋은 방법이 있는지, 현재 코드가 최선인지 피드백해주세요.

- 비유를 많이 사용하여 이해하기 쉽게 설명하세요

# 57. 설정 파일 (settings.json)
- .claude/settings.json
  - 팀 공유 설정
- .claude/settings.local.json
  - 개인 선호도에 따른 설정
- 설정 우선순위
  1. 관리 정책 설정 -> 클로드 코드 내부
  2. 개인 설정 -> .claude/settings.local.json
  3. 프로젝트 설정 -> .claude/settings.json
  4. 사용자 설정 -> ~/.claude/settings.json
- 자주 사용하는 것 -> 디폴트로 설정해도 좋다.
```json
{
    "permissions": {
        "allow": [
            "Bash"
        ],
        "deny": [],
        "ask": [],
        "defaultMode": "plan",
        "additinalDirectories": ["폴더명"]
    },
    "outputStyle": "Explanatory"
}
```
- API key, 환경 파일 등 민감한 정보를 Claude Code가 접근하지 못하도록 설정
```json
{
  "permissions": {
    "deny": [
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./.secrets/**)",
      "Read(./.config/credentials.json)",
      "Read(./build/**)"
    ]
  }
}
```
- 파일이나 폴더를 읽지 않도록 설정 가능! -> 클로드 코드가 접근하지 못하도록 할 때 사용

# 59. 메모리 관리: CLAUDE.md
- 메모리 파일 -> CLAUDE.md
  - 프로젝트에서 지켜야할 규칙을 적어놓는 구조
  - 클로드코드가 매번 세션 시작시, 자동으로 읽는다.
  - /init -> CLAUDE.md 가이드로 프로젝트 초기화
- 메모리 파일 종류
  - 어디에 위치시키느냐에 따라 역할이 다르다.
  - ./CLAUDE.md || ./.claude/CLAUDE.md: 프로젝트 메모리(프로젝트를 위한 팀 공유 지침)
  - ./.claude/rules/*.md: 프로젝트 규칙(모듈식, 주제별 프로젝트 지침)
  - ~/.claude/CLAUDE.md: 개인 메모리(공통 개인 지침)
  - ./CLAUDE.local.md: 프로젝트 메모리(로컬, 개인 규칙 관리)
- 메모리 파일 길어질 경우 -> 누락될 수도 있다.
  - IMPORTANT, YOU MUST 등 강조 문구를 통해 준수도를 향상시켜야 한다.
- CLAUDE.md
  - 공식 문서 -> 500줄 이하로 유지 권장
    - 중복 최소화
    - 메모리 효율 관리 -> @path/to/import -> 특정 파일을 메모리로 읽는다.
      - 파일 import를 통해, 파일 라인을 줄이면서 원하는 지침 추가 가능
- 메모리 파일
  - 구체적 작성 필요
    - [X] 코드를 적절히 포맷합니다. -> 모호해서 클로드 멋대로 판단 
    - [O] 두 칸 들여쓰기 사용 
  - 구조를 사용해 구성
    - List 형식으로 그룹화 -> 더 잘 이해하고 준수할 확률 늘어난다.
  - 정기적으로 CLAUDE.md 업데이트
    - 처음부터 무리해서 작성 X
    - 프로젝트 진화함에 따라 함께 수정
  - 누락 가능성 -> 중요한 것은 [중요] 등으로 표시
- 명령어
  - /memory
    - 로드된 메모리 확인 및 편집 가능

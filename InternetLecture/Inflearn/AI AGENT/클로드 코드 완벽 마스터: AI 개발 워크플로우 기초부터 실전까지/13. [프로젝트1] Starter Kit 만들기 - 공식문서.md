# 74. 오류 수정과 명확한 프롬프트 엔지니어링
- claude code
```text
당신은 웹 개발 전문가입니다.
현재 웹 애플리케이션에는 404 오류가 발생하고 있어요.
헤더 메뉴를 클릭하면 404 오류가 발생을 하는 메뉴가 있고요,
"예제" 메뉴는 진입이 되지만 그 안에서 버튼링크를 클릭시 오류가 발생해요!
이를 해결해주세요.
단계 별로 생각해주세요.
1. 헤더 메뉴에서 필요없는 메뉴는 제거 "기능", "가격" (먼저 제거를 진행하세요)
2. 웹 애플리케이션의 404 오류 목록을 수집해주세요.
3. 오류를 수정해주세요.
```
- 공식 문서에서도 작업 사항을 명확하고 직접적이며 상세하게 작성하라고 가이드!
  - 작업 맥락을 최소한으로 알고 있는 사람에게 물어봐라. -> 혼란스러워한다면 클로드도 마찬가지다.
  - 프롬프트
    - 작업 결과 -> 어디에 사용되는지
    - 출력물 -> 대상이 누군지
    - 작업 -> 어떤 워크플로우의 일부인지, 해당 워크 플로우에서 어디에 속하는지
    - 작업 -> 최종 목표, 성공적인 작업 완료가 어떤 모습인지
- 다소 복잡한 작업 -> 한번에 요청 -> 많은 오류가 발생할 수 있다.
  - 복잡한 작업을 끊어서 요청
- 브라우저 테스트 자동화 필요!

# 75. Playwright MCP란? - 오류 해결하기
- 웹 -> 클로드가 직접 브라우저 접속 -> 오류 발생 파악 -> 직접 수정 flow
- Playwright
  - 웹 브라우저 자동화 도구
  - 모든 브라우저 제어 -> 웹 개발시 테스트 자동화, 크롤링, 자동 화면 캡쳐 -> 브라우저 제어
- 클로드 -> 웹 사이트 접속 후 수집 -> 클로드가 접속 후 원하는 정보 수집
- Playwright MCP 설치
  - claude mcp add playwright npx @playwright/mcp@latest --scope project
  - mcp__github
  1. 오류 정보 수집
  2. 수집 오류 바탕으로 오류 원인 분석
  3. 오류 해결 -> 한 번에 해결 안 될 수도
  4. 테스트 -> 오류 발견시 1번부터 다시 순환
